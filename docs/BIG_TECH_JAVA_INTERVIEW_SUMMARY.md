### 1. 京东的典型特点

- 很多面经会从**项目经历**切入；
- 喜欢问“如果业务量大 / 并发高 / 风险高，你会怎么设计”；
- Redis、锁、MQ、事务、一致性、重构类问题比较常见；
- 如果岗位偏 AI 应用 / Agent / 业务平台，面试官很可能会追问：
  - 你怎么做隔离？
  - 怎么防误操作？
  - 怎么做审批、回滚、幂等、可观测？

### 2. 字节跳动的典型特点

- 非常看重**基础是否真正理解**，不喜欢“背答案感”；
- 同一个点会连续追问：
  - 是什么？
  - 为什么？
  - 怎么实现？
  - 缺点是什么？
  - 线上怎么选型？
- 常见表现是：
  - `HashMap` -> 扩容 / 冲突 / 线程安全 / `ConcurrentHashMap`
  - `MySQL` -> 隔离级别 / 索引 / B+Tree / 联合索引 / 回表 / 覆盖索引
  - `IO` -> `select` / `poll` / `epoll`
  - `JVM` -> 内存结构 / GC / 垃圾回收器 / 对象分配

### 3. 阿里的典型特点

- 问题覆盖面广，喜欢从**Java 基础 -> 框架 -> 中间件 -> 分布式 -> 网络**一路铺开；
- 很多问题不是单点，而是考你是否形成了完整知识体系；
- 常见表现是：
  - `JMM` / `volatile` / CAS / 锁 / 乐观锁
  - `Spring Boot` 自动装配 / IoC / AOP / 动态代理
  - `Zookeeper` / 一致性 / 脑裂
  - `Kafka` / 消息可靠性 / 顺序 / 重复消费
  - `CDN` / `DNS` / `SSL` / `NAT` / 网络链路分析

## 三、后端 Java 通用高频题总表

下面这些题，不是某一家独有，而是三家大概率都会从中抽样问。

### 1. Java 基础与集合

#### 高频题 1：`HashMap` 底层实现是什么？1.7 和 1.8 有什么区别？

**答题要点：**

- 底层是数组 + 链表 / 红黑树；
- `hash` 冲突通过链地址法处理；
- 1.8 在链表长度超过阈值且数组容量足够时树化；
- 1.7 扩容头插法在并发下可能形成环；1.8 做了改进；
- 非线程安全，多线程场景下用 `ConcurrentHashMap`。

#### 高频题 2：`ArrayList` 和 `LinkedList` 区别？

**答题要点：**

- `ArrayList` 查询快、尾插快、随机访问友好；
- `LinkedList` 插入删除理论上灵活，但实际局部性差、遍历成本高；
- 大多数业务场景优先 `ArrayList`。

#### 高频题 3：`ConcurrentHashMap` 为什么更适合并发场景？

**答题要点：**

- 1.8 主要是 `CAS + synchronized + volatile`；
- 不是整表加锁，锁粒度更细；
- 读操作尽量无锁；
- 需要会讲到扩容协助、桶位竞争、可见性。

### 2. 并发与 JUC

#### 高频题 4：`synchronized` 和 `ReentrantLock` 区别？

**答题要点：**

- `synchronized` 是 JVM 层关键字，使用简单；
- `ReentrantLock` 功能更强：可中断、可超时、可公平锁、多个 `Condition`；
- 高并发或复杂等待队列场景更常用 `ReentrantLock`；
- 本质都依赖同步机制，但使用体验和扩展性不同。

#### 高频题 5：AQS 是什么？

**答题要点：**

- `AbstractQueuedSynchronizer`，JUC 同步器基础框架；
- 核心是一个 `state` 状态 + CLH 变体等待队列；
- `ReentrantLock`、`Semaphore`、`CountDownLatch` 等都依赖 AQS；
- 获取锁失败入队、释放锁唤醒后继节点。

#### 高频题 6：CAS 是什么？有什么问题？

**答题要点：**

- 比较并交换，无锁原子操作；
- 典型问题：ABA、自旋开销大、只能保证一个共享变量原子性；
- ABA 可通过版本号处理，如 `AtomicStampedReference`。

#### 高频题 7：乐观锁和悲观锁怎么选？

**答题要点：**

- 冲突少用乐观锁，吞吐更高；
- 冲突多、失败重试成本高用悲观锁；
- 数据库里乐观锁常用版本号 / 时间戳；
- 业务上还要结合幂等、补偿、隔离级别看。

### 3. JVM

#### 高频题 8：JVM 内存结构怎么分？

**答题要点：**

- 线程私有：程序计数器、虚拟机栈、本地方法栈；
- 线程共享：堆、方法区（元空间）；
- 对象主要在堆中，类元数据在元空间；
- 栈里主要是局部变量表、操作数栈、栈帧。

#### 高频题 9：常见 GC 算法和垃圾回收器？

**答题要点：**

- 算法：标记-清除、复制、标记-整理、分代收集；
- 回收器：Serial、ParNew、CMS、G1；
- 需要能讲 G1 的分区思想、停顿目标、适用场景。

#### 高频题 10：对象什么时候进入老年代？

**答题要点：**

- 年龄阈值；
- 大对象直接进入老年代；
- Survivor 放不下时动态晋升；
- 晋升失败可能触发 Full GC。

### 4. Spring / Spring Boot

#### 高频题 11：Spring IoC 和 AOP 是什么？

**答题要点：**

- IoC：对象创建和依赖管理由容器负责；
- DI 是实现手段；
- AOP 用于把横切关注点抽离，如日志、事务、鉴权；
- 动态代理实现：JDK 动态代理 / CGLIB。

#### 高频题 12：Spring Boot 自动装配原理？

**答题要点：**

- 启动类触发自动配置；
- 从自动配置导入机制装载候选配置类；
- 借助条件注解按类路径、Bean、配置项等决定是否生效；
- 重点是“约定优于配置，但仍可覆盖”。

#### 高频题 13：Spring 事务失效的场景有哪些？

**答题要点：**

- 同类自调用；
- 方法不是 `public`；
- 异常被吞掉；
- 事务传播行为配置不当；
- 数据源 / 代理对象没有生效。

### 5. MySQL

#### 高频题 14：事务四大特性和隔离级别？

**答题要点：**

- ACID；
- 隔离级别：读未提交、读已提交、可重复读、串行化；
- 脏读、不可重复读、幻读概念要能区分；
- InnoDB 默认 `REPEATABLE READ`。

#### 高频题 15：为什么索引用 B+Tree，不用 B Tree / 红黑树 / 哈希？

**答题要点：**

- 磁盘 / 页读取模型友好；
- 非叶子节点只存索引，扇出更大，层数更低；
- 叶子节点天然适合范围查询和顺序遍历；
- 红黑树层高高，哈希不支持范围查询。

#### 高频题 16：联合索引最左前缀原则怎么理解？

**答题要点：**

- 按索引定义顺序匹配；
- 中间断了，后面的列不能充分利用；
- 注意范围查询、排序、回表、覆盖索引的影响；
- 面试常追问：`where a=? and c=?` 为什么不如 `a,b,c` 全命中。

#### 高频题 17：什么情况下会索引失效？

**答题要点：**

- 对索引列做函数 / 计算；
- 隐式类型转换；
- 前导 `%` 模糊查询；
- 不符合最左匹配；
- 选择性太差导致优化器不走索引。

### 6. Redis

#### 高频题 18：Redis 常见数据结构及使用场景？

**答题要点：**

- `String`：缓存、计数器、分布式锁；
- `Hash`：对象缓存；
- `List`：消息队列简单场景；
- `Set`：去重；
- `ZSet`：排行榜、延时任务；
- 需要能讲底层编码和大 key 风险更好。

#### 高频题 19：缓存一致性怎么做？

**答题要点：**

- 经典思路是先更新数据库，再删除缓存；
- 结合延迟双删、消息通知、重试补偿；
- 关键关注：并发读写竞争、失败补偿、幂等、热点 key。

#### 高频题 20：Redis 分布式锁注意什么？

**答题要点：**

- `SET key value NX PX ttl`；
- 解锁要校验 value；
- 注意锁超时、业务执行过长、误删他人锁；
- 对“强一致”场景不要盲目迷信分布式锁。

### 7. MQ / 分布式

#### 高频题 21：如何保证消息可靠性？

**答题要点：**

- 生产端：发送确认、重试、事务消息（看中间件能力）；
- Broker：持久化、副本机制；
- 消费端：手动 ack、失败重试、死信队列；
- 业务端：幂等、防重复消费。

#### 高频题 22：消息重复消费怎么办？

**答题要点：**

- 幂等键；
- 去重表 / 去重缓存；
- 状态机判重；
- 业务接口本身设计为幂等。

#### 高频题 23：分布式事务怎么处理？

**答题要点：**

- 强一致通常代价高；
- 互联网业务常用最终一致性；
- 方案：本地消息表、事务消息、TCC、Saga；
- 重点不是背概念，而是能说清业务权衡。

### 8. 网络 / 操作系统

#### 高频题 24：从浏览器输入 URL 到页面展示发生了什么？

**答题要点：**

- URL 解析；
- DNS 查询；
- TCP 三次握手；
- TLS 握手；
- HTTP 请求转发；
- 反向代理 / 网关 / 应用服务；
- 数据库 / 缓存访问；
- 响应返回与页面渲染。

#### 高频题 25：`select`、`poll`、`epoll` 区别？

**答题要点：**

- `select` 有 fd 数量限制，位图拷贝成本高；
- `poll` 解决数量限制，但仍要轮询；
- `epoll` 事件驱动，更适合高并发连接；
- 继续追问时要能说水平触发 / 边缘触发。

### 9. 项目与系统设计

#### 高频题 26：如果系统并发量突然升高，你会从哪些角度处理？

**答题要点：**

- 限流；
- 缓存；
- 异步化；
- 削峰填谷；
- 读写分离 / 分库分表；
- 热点隔离；
- 线程池、连接池参数；
- 可观测性与压测。

#### 高频题 27：如果让你设计一个高可用下单 / 审批 / 任务系统，你会怎么做？

**答题要点：**

- 先讲业务流程和状态机；
- 再讲幂等、事务边界、消息驱动；
- 再讲缓存、降级、超时、补偿；
- 最后讲监控、审计、回滚、权限。

## 四、京东后端 Java 面试题与面经整理

### 1. 京东高频考察画像

从公开面经看，京东后端 Java 岗位的特点是：

- 不是只看八股，更看你**能不能把项目说成一个真实系统**；
- 喜欢围绕：
  - 高并发
  - 锁
  - Redis
  - RocketMQ / 消息可靠性
  - 重构旧系统
  - 业务风险控制
  - 多人协作 / 项目推进
- 如果岗位带 AI / Agent / 业务自动化属性，会问：
  - 如何避免大模型误操作真实业务？
  - 如何做沙箱、审批、隔离、回滚？
  - 如何做上下文、记忆、日志追踪？

### 2. 京东常见面试流程（按公开面经归纳）

常见是：

1. 一面：技术面 + 项目面；
2. 二面：继续深挖项目、业务理解、设计能力；
3. 少数岗位会有三面 / leader 面 / HR 面。

一面的典型风格：

- 自我介绍 + 项目经历；
- 基础题穿插提问；
- 问你做过什么，而不是只问你背过什么。

二面的典型风格：

- 场景题明显增加；
- 关注你是否有“工程判断力”；
- 关注你遇到复杂历史系统时如何推进改造。

### 3. 京东高频题目清单

#### 并发与锁

- `ReentrantLock` 底层实现是什么？
- 锁的基本原理是什么？
- `synchronized` 和其他并发控制方式怎么选？
- 并发量高的系统，写代码时需要考虑哪些问题？

**答题方向：**

- 共享资源识别；
- 锁粒度控制；
- 避免长任务和大锁；
- 限流、缓存、异步、批量化；
- 线程池和队列治理；
- 监控和压测验证。

#### Redis

- Redis 基本数据结构有哪些？
- Redis 事务适合什么场景？
- 你的业务里 Redis 怎么用？
- 缓存穿透、击穿、雪崩怎么处理？

#### MQ

- RocketMQ 怎么保证消息可靠性？
- 消息重复消费怎么办？
- 顺序消息什么时候需要，代价是什么？

#### Spring / Java / 框架

- IoC / AOP 原理；
- Java 常用数据结构；
- 多线程问题如何避免；
- 项目里框架分层如何设计。

#### 项目与场景题

- 金融系统不允许 Agent 直接执行真实操作，你怎么设计？
- 重构历史业务时又来了新需求，怎么处理？
- 多人协作项目中，你如何拆分边界与推进落地？

#### 面经 A：`5.18京东一面`

#### 面经 B：`后端Agent开发一面分享-京东实习`

- 来源：<https://www.nowcoder.com/feed/main/detail/2848492>
- 关键词：Agent / 金融系统 / Java 数据结构
- 归纳：
  - 非常典型的业务安全场景题：如果系统不能允许 Agent 直连真实交易动作，你如何设计隔离层；
  - 还会穿插 Java 基础，如常用数据结构。
- 复习建议：
  - 要会答“沙箱 + 审批 + 风控规则 + 人工确认 + 操作审计 + 回滚补偿”；

#### 面经 C：`暑期实习 京东后端Agent开发一面面经`

- 来源：<https://www.nowcoder.com/feed/main/detail/2845451>
- 关键词：高并发 / `ReentrantLock` / 数据结构
- 归纳：
  - 关注高并发代码设计；
  - 追问锁、`ReentrantLock` 底层实现；
  - 这是典型的“业务 + 并发原理”组合拳。
- 复习建议：
  - 不要只会说“加锁”；

## 七、答题模板：技术题到底怎么答

很多人不是不会，而是答得乱。建议统一用下面模板。

### 模板 1：定义类问题

适合：`HashMap`、AOP、事务隔离级别、CAS 这类题。

回答顺序：

1. 先一句话定义；
2. 再讲核心原理；
3. 再讲优缺点；
4. 再讲业务中的使用场景；
5. 最后补一个边界 / 风险点。

### 模板 2：场景设计题

适合：高并发、缓存一致性、系统重构、Agent 风控。

回答顺序：

1. 先确认业务目标；
2. 再拆核心风险；
3. 再给主方案；
4. 再补异常和兜底；
5. 最后讲监控与验证。

### 模板 3：项目深挖题

适合：介绍项目、介绍你做的模块。

回答顺序：

1. 背景；
2. 目标；
3. 难点；
4. 方案；
5. 结果；
6. 反思。

不要一上来就堆技术名词。

## 十、最后的准备建议

如果你现在时间不多，优先级建议如下：

### 第一优先级

- Java 并发
- JVM
- MySQL
- Redis
- Spring / Spring Boot

### 第二优先级

- MQ
- 分布式一致性
- 网络链路
- 项目深挖

### 第三优先级

- 手撕代码
- Linux / OS 细节
- 特定业务线知识

一句话总结：

- **面京东：项目要真，场景要实，工程要落地。**
- **面字节：基础要深，追问要扛，代码要稳。**
- **面阿里：体系要全，原理要通，分布式和网络别太弱。**
